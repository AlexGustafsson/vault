# `audit enable`

Enable an audit device at the given path or return an error if a device already
exists.

## Syntax and usage

<CodeBlockConfig hideClipboard>

```text
vault audit enable <type> [flags] [args]
```

</CodeBlockConfig>

Supported audit device types:

- `file` - write and append audit logs to a local file.
- `syslog` - stream audit logs to the local syslog agent.
- `socket` - stream audit logs to a TCP, UDP, or UNIX socket.

## Examples

Enable the audit device "file" at the path "/tmp/my-file.txt":

```shell-session
$ vault audit enable file file_path=/tmp/my-file.txt
Success! Enabled the file audit device at: file/
```
