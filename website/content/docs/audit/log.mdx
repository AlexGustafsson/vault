# Audit Log Device

The goal of this document is to recommend _HashiCorp Vault_ Syslog Logging and Audit Logging configuration and notable events to create alerts for. Vault has two types of logs: syslog and audit logs. Syslogs are general high-level service logs where common global actions are recorded, such as service startup, shutdown, and configuration changes. The Audit Log contains fine-grained access, action, and security logs.

## Systemd Service Configuration

[Vault Deployment Guide - Step 3: Configure systemd](https://learn.hashicorp.com/vault/operations/ops-deployment-guide#step-3-configure-systemd)

The Vault systemd service file is located at: `/lib/systemd/system/vault.service`. Ensure that the `ExecStart` parameter configuration command does not contain a `>` character, which would redirecting STDOUT and STDERR into a file and may interferre with logging.

```
[Unit]
Description=Vault Server
Requires=consul-online.target
After=consul-online.target

[Service]
Restart=on-failure
EnvironmentFile=/etc/vault.d/vault.conf
PermissionsStartOnly=true
ExecStartPre=/sbin/setcap 'cap_ipc_lock=+ep' /usr/local/bin/vault
ExecStart=/usr/local/bin/vault server -config /etc/vault.d $FLAGS
ExecReload=/bin/kill -HUP $MAINPID
KillSignal=SIGTERM
User=vault
Group=vault
LimitMEMLOCK=infinity

[Install]
WantedBy=multi-user.target
```

_systemd commands_

```
# enable the systemd HashiCorp Vault service
sudo systemctl enable vault

# start the systemd HashiCorp Vault service
sudo systemctl start vault

# check the status of the HashiCorp Vault service
sudo systemctl status vault
```

Common commands to search _HashiCorp Vault_ logs.

```
grep -F 'vault[' /var/log/syslog

journalctl -b --no-pager -u vault -o json | jq '.MESSAGE'

cat /var/log/vault_audit.log | jq '.'
```

Check if any Vault Audit devices are enabled:

`vault audit list -detailed`

_cli response_

```
No audit devices are enabled.
```

## Server Configuration

_HashiCorp Vault_ can be run in development mode by setting configuration in the file `/etc/vault.d/vault.conf`

```
FLAGS=-dev -dev-ha -dev-transactional -dev-root-token-id=root -dev-listen-address=0.0.0.0:8200
```

Vault Syslog Logging and Audit Logging can be enabled in the default Vault configuration file: `/etc/vault.d/vault.hcl`

## Audit Log, File

[Vault - File Audit Device](/docs/audit/file)

Run these commands for initial setup:

```
# create the HashiCorp Vault Audit Log file
touch /var/log/vault_audit.log

# set permissions on the HashiCorp Vault Audit Log file
chown vault:vault /var/log/vault_audit.log

# clear out the HashiCorp Vault Audit Log file
echo "" > /var/log/vault_audit.log
```

## Audit Log, Syslog

[Vault - Syslog Audit Device](/docs/audit/syslog)

Run these commands for initial setup:

```
# create the journald folder
mkdir -p /var/log/journal
```

If you want the syslog to be stored in a file on disk, on Ubuntu 16.04, the journald configuration is stored at `/etc/systemd/journald.conf` Set the configuration:

```
Storage=persistent
```

Now restart the systemd journald service with `systemctl force-reload systemd-journald`

### enable Vault audit log, syslog

`vault audit enable syslog tag="vault" facility="AUTH"`

_cli response_

```
Success! Enabled the syslog audit device at: syslog/
```

You can verify HashiCorp Vault is logging to syslog with these commands:

```
# check that HashiCorp Vault Audit is logging to journald
journalctl -b --no-pager -u vault

# read the HashiCorp Vault Audit Log as JSON
journalctl -b --no-pager -u vault -o json | jq '.'

# save the HashiCorp Vault Audit Log into a compressed file
journalctl -b --no-pager -u vault | gzip -9 > /tmp/"$(hostname)-$(date +%Y-%m-%dT%H-%M-%SZ)-vault.log.gz"
```

### check Vault Audit Log, Syslog, is enabled

`vault audit list -detailed`

_cli response_

```
Path       Type      Description    Replication    Options
----       ----      -----------    -----------    -------
syslog/    syslog    n/a            replicated     facility=AUTH tag=vault
```

### disable Vault Audit Log, Syslog

`vault audit disable syslog`

_cli response_

```

```

_syslog_

